# 页面子包

日期：2026-03-10

## 用法

| 项 | 内容 |
|---|---|
| 文档角色 | 页面级实现拆解包 |
| 目标 | 告诉外部 AI 每个页面该由什么组件组成、吃哪些接口、必须处理哪些状态 |
| 搭配关系 | 必须和 [`前端包.md`](./前端包.md) 一起用 |

## 全局页面约束

| 项 | 要求 |
|---|---|
| 页面总数 | 仅限 7 页：登录、首页、车辆、记录、知识、商品、预约 |
| 数据来源 | 全部以后端现役接口为准，不允许前端自行拼业务真相源 |
| 状态分支 | 每页至少考虑 `loading`、`success`、`empty`、`error`；涉及聚合模块的页面还要处理 `hidden/conflict/unavailable` |
| 组件拆分 | 优先按 `TRD` 中的组件 contract 拆，不要一个页面塞成一个超大组件 |
| 文案基调 | 功能清晰、提示明确、避免营销腔 |

## 登录页

| 维度 | 说明 |
|---|---|
| 页面目标 | 让用户默认使用演示账号快速进入首页，同时保留手动修改能力 |
| 主要组件 | `LoginForm` |
| 主要接口 | `POST /api/auth/login` |
| 核心字段 | `username`、`password`、返回 `token` 与 `user` |
| 默认值 | 首次渲染预填 `username=admin`、`password=123456` |
| 必做状态 | `loading`、登录失败、密码为空前端拦截、默认账号一键提交 |
| 页面结果 | 成功后保存 token，跳转首页 |
| 文件落点建议 | `pages/Login*`、`components/LoginForm*` |

## 首页

| 维度 | 说明 |
|---|---|
| 页面目标 | 展示当前车辆总览，并给出下一步动作 |
| 主要组件 | `VehicleSwitcher`、`HealthCard`、`SpecCard`、`ProductStrip`、`KnowledgeStrip`、`ButlerEntry` |
| 主要接口 | `GET /api/home/vehicles`、`GET /api/home/overview` |
| 核心字段 | `currentVehicle`、`tireHealth`、`recommendedProducts`、`weeklyKnowledge`、`modules` |
| 必做状态 | `loading`、无车空态、模块 `hidden`、模块 `conflict`、接口失败 |
| 页面结果 | 用户能切车、看健康状态、看推荐商品和知识、进入下一步 |
| 文件落点建议 | `pages/Home*`、`components/home/*` |

## 车辆页

| 维度 | 说明 |
|---|---|
| 页面目标 | 展示单车基础信息、健康状态和服务卡提醒 |
| 主要组件 | `VehicleSummary`、`HealthBadge`、`ServiceCardBanner` |
| 主要接口 | `GET /api/vehicles/{id}`、`GET /api/vehicles/{id}/health`、`GET/PATCH /api/vehicles/{id}/service-card*` |
| 核心字段 | `licensePlate`、`modelName`、`currentMileage`、`tireSpec`、`ServiceCardStatus` |
| 必做状态 | `loading`、车辆不存在、`HEALTHY` 隐藏提醒、`REPLACE/ROTATE` 显示提醒、ack 后隐藏 |
| 页面结果 | 用户知道“这台车当前该不该处理” |
| 文件落点建议 | `pages/Vehicle*`、`components/vehicle/*` |

## 记录页

| 维度 | 说明 |
|---|---|
| 页面目标 | 查看、添加、编辑、删除轮胎记录 |
| 主要组件 | `RecordList`、`RecordFormDialog` |
| 主要接口 | `GET /api/tire-records/vehicle/{vehicleId}`、`POST/PUT/DELETE /api/tire-records*` |
| 核心字段 | `date`、`mileage`、`count`、`type`、`vehicleId` |
| 必做状态 | `loading`、无记录空态、提交失败、删除确认 |
| 页面结果 | 用户能维护这台车的轮胎历史 |
| 文件落点建议 | `pages/Records*`、`components/records/*` |

## 知识页

| 维度 | 说明 |
|---|---|
| 页面目标 | 按分类浏览轮胎知识 |
| 主要组件 | `KnowledgeTabs`、`KnowledgeCardList` |
| 主要接口 | `GET /api/knowledge?category=` |
| 核心字段 | `type`、`title`、`content`、`category`、`thumbnail` |
| 必做状态 | `loading`、分类切换、无内容空态、接口失败 |
| 页面结果 | 用户能按分类消费知识内容 |
| 文件落点建议 | `pages/Knowledge*`、`components/knowledge/*` |

## 商品页

| 维度 | 说明 |
|---|---|
| 页面目标 | 浏览商品、查看详情、进入预约动作 |
| 主要组件 | `ProductFilterBar`、`ProductCardList`、`ProductDetailHero` |
| 主要接口 | `GET /api/products`、`GET /api/products/{id}`、`GET /api/purchase/detail` |
| 核心字段 | `brand`、`name`、`spec`、`price`、`originalPrice`、`minPrice`、`maxPrice`、`matchedSpecCode`、`modules` |
| 必做状态 | `loading`、无商品空态、规格冲突、详情模块隐藏、接口失败 |
| 页面结果 | 用户能完成“看商品 -> 去预约”的动作闭环 |
| 文件落点建议 | `pages/Product*`、`components/product/*` |

## 预约页

| 维度 | 说明 |
|---|---|
| 页面目标 | 提交预约并展示结果 |
| 主要组件 | `AppointmentForm`、`SubmitResult` |
| 主要接口 | `POST /api/appointments`、`GET /api/appointments/{id}` |
| 核心字段 | `vehicleId`、`mode`、`storeId`、`note`、`sourcePage` |
| 必做状态 | `loading`、提交成功、提交失败、非法 mode、门店不存在 |
| 页面结果 | 用户能清楚知道预约是否创建成功 |
| 文件落点建议 | `pages/Appointment*`、`components/appointment/*` |

## 页面与车辆故事映射

| 页面 | 重点样本 | 用途 |
|---|---|---|
| 首页 | 车 1、车 2、车 3、车 5 | 验证健康态、服务卡、推荐商品、知识模块差异 |
| 车辆页 | 车 1、车 3、车 6 | 验证 `HEALTHY` 隐藏、`REPLACE` 提醒、AI 失败不串主线 |

| 预约页 | 车 1、车 2、车 5、车 6 | 验证 `PICKUP/SELF`、来源页、备注文案 |
