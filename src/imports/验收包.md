# 验收包

日期：2026-03-10

## 用法

| 项 | 内容 |
|---|---|
| 文档角色 | 前端外发验收标准包 |
| 目标 | 限定外部 AI 交付什么、验证什么、做到什么算完成 |
| 搭配关系 | 必须和 [`前端包.md`](./前端包.md) 及 [`页面子包.md`](./页面子包.md) 一起用 |

## 交付物清单

| 类别 | 必须交付 |
|---|---|
| 代码 | 可落地的前端代码，不接受只给思路或伪代码 |
| 文件清单 | 明确新增/修改哪些文件 |
| 说明 | 关键改动说明、假设说明、未完成项说明 |
| 运行方式 | 如何启动前端并联调后端 |
| 验证结果 | 至少给出浏览器验收说明；如果可行，附截图 |

## 全局验收标准

| 验收项 | 通过标准 |
|---|---|
| 字段一致性 | 不得发明平行字段，字段名必须与后端和真库口径一致 |
| 接口一致性 | 请求路径、请求体、响应处理符合 `TRD` |
| 模块状态 | 涉及 `FrontendShell` 的页面必须正确消费 `modules` |
| 错误分支 | 不能只做 happy path，必须有 empty/error 分支 |
| 边界遵守 | 不扩后台、不扩支付、不改数据库治理口径 |
| 可验证性 | 必须说明如何在浏览器中验证关键链路 |

## 页面级验收

| 页面 | 必验项 |
|---|---|
| 登录页 | 默认填充 `admin / 123456`、允许手动修改、空值拦截、登录失败提示、登录成功跳转 |
| 首页 | 车辆切换、健康卡、规格卡、推荐商品、每周知识、模块显隐 |
| 车辆页 | 单车信息、健康状态、服务卡显示/隐藏、ack 后结果 |
| 记录页 | 记录列表、新增、编辑、删除、无记录空态 |
| 知识页 | 分类切换、内容列表、空态、失败态 |
| 商品页 | 商品列表、规格展示、详情入口、模块隐藏/冲突处理 |
| 预约页 | `PICKUP/SELF` 提交成功、失败提示、结果页展示 |

|---|---|

## 浏览器验收要求

| 项 | 要求 |
|---|---|
| 验证方式 | 优先真实浏览器验收，不接受只说“代码应该能跑” |
| 验证内容 | 至少覆盖登录、首页、一个详情页、一个表单提交页 |
| 截图要求 | 如果工具支持浏览器截图，至少给首页和预约结果页 |
| 降级说明 | 如果无法截图，必须明确说明原因和替代验证方式 |

## 不通过的常见情况

| 情况 | 判定 |
|---|---|
| 只给组件草图，没有可运行代码 | 不通过 |
| 登录页没有默认填充演示账号 `admin / 123456` | 不通过 |
| 把 `modules` 完全忽略 | 不通过 |
| 自己发明 `carId`、`source` 之类的平行字段 | 不通过 |
| 预约 mode 写成 `INSPECT/REPAIR` 可直接提交 | 不通过 |
| 看起来漂亮，但没有空态和失败态 | 不通过 |
