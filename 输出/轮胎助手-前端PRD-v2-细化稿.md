# 轮胎助手前端 PRD v2 细化稿

## 1. 文档说明
- 本版在 v1 基础上继续细化页面结构、状态分支、交互事件、数据依赖与验收规则
- 文档基于当前仓库实现反推，不代表真实后端已经完成
- 文档目的不是“描述想法”，而是让任何接手人可以按文档继续写前端、补后端、做设计、拆任务

## 2. 产品范围重述
### 2.1 主流程
1. 用户进入登录页
2. 使用默认演示账号快速登录
3. 进入首页查看车辆与轮胎状态
4. 从首页进入车辆详情、知识、AI 识别、商品、预约或记录
5. 完成“看状态 -> 看商品 -> 去预约”的闭环

### 2.2 子流程
- 车辆切换
- 规格查看/调整
- AI 磨损识别
- 商品筛选与详情查看
- 预约方式选择与提交
- 换胎记录维护
- 门店搜索与选择

## 3. 角色、目标、成功判定
### 3.1 角色
- 车主用户（唯一主角色）

### 3.2 车主目标
- 知道车辆轮胎当前是否正常
- 知道当前轮胎规格
- 在异常时快速找到处理方式
- 能记录历史维护行为

### 3.3 成功判定
- 用户不离开首页也能理解下一步
- 用户进入任何功能页后，页面结构和动作明确
- 用户在 3 次以内点击中能到达预约页

## 4. 技术与框架约束
- 前端框架: React 18
- 路由: `react-router`
- 样式: Tailwind 风格 class + 自定义 CSS
- 构建工具: Vite
- 数据来源: 当前为 `mock.ts`
- 数据字段真源要求:
  - `vehicleId`
  - `userId`
  - `storeId`
  - `sourcePage`
  - `createdAt`
  - `updatedAt`
  - `spec`
  - `matchedSpecCode`

## 5. 路由规格
### 5.1 路由清单
| 编号 | 路由 | 页面 | 入口 |
|---|---|---|---|
| RT-001 | `/login` | 登录页 | 独立入口 |
| RT-002 | `/home` | 首页 | 登录成功后 |

| RT-006 | `/products` | 商品列表页 | 首页、底部商城 |
| RT-007 | `/products/:id` | 商品详情页 | 首页推荐、列表卡片 |
| RT-008 | `/appointment` | 预约页 | 首页、商品详情、车辆页 |
| RT-009 | `/records` | 记录页 | 首页健康卡、底部导航 |

### 5.2 路由共同行为
- 每次路由切换自动滚动到顶部
- 部分页面依赖路由参数:
  - `:id`

  - `vehicleId`
  - `source`

## 6. 全局组件规格
### 6.1 Header
- 输入:
  - `title`
  - `showBack`
  - `showBell`
- 行为:
  - `showBack=true` 时左侧展示返回按钮
  - 点击返回按钮执行 `navigate(-1)`
  - `showBell=true` 时右侧显示消息按钮

### 6.2 BottomNav
- 固定底部展示
- 五个入口固定顺序:
  - 首页
  - 知识
  - 预约
  - 商城
  - 记录
- 当前路由匹配项高亮

### 6.3 ProductVisual
- 用品牌色和轮胎抽象图呈现商品视觉
- 非真实商品图，属于前端表现层包装组件
- `compact=true` 时用于列表/推荐小卡

## 7. 数据模型与业务含义
### 7.1 健康状态
| 值 | 含义 | 前端表现 |
|---|---|---|
| `HEALTHY` | 状态正常 | 绿色 |
| `ROTATE` | 建议换位 | 黄色 |
| `REPLACE` | 建议更换 | 红色 |

### 7.2 预约模式
| 值 | 含义 |
|---|---|
| `PICKUP` | 上门取送车 |
| `SELF` | 自行到店 |

### 7.3 模块状态
| 值 | 含义 | 处理原则 |
|---|---|---|
| `ready` | 正常显示 | 直接渲染 |
| `hidden` | 不应显示 | 页面隐藏该模块 |
| `conflict` | 模块冲突 | 呈现错误/冲突提示 |
| `unavailable` | 暂不可用 | 呈现降级提示 |

### 8.2 首页
#### 8.2.1 页面目标
- 呈现“当前车辆当前状态 + 最近建议动作”

#### 8.2.2 结构拆解
- HOME-A: 顶部品牌 Header
- HOME-B: 车辆切换卡片
- HOME-C: 轮胎健康度卡片
- HOME-D: 轮胎知识卡片
- HOME-E: 轮胎小管家卡片
- HOME-F: AI 识别入口卡片
- HOME-G: 商品推荐卡片
- HOME-H: 底部导航

#### 8.2.3 本地状态
- `currentVehicleIndex`

#### 8.2.4 计算逻辑
- `currentVehicle = MOCK_VEHICLES[currentVehicleIndex]`
- `modules = VEHICLE_MODULES[currentVehicle.vehicleId]`
- `recommendedProducts = 按 matchedSpecCode 过滤后取前 2`
- `weeklyKnowledge = 知识数组前 3 条`
- `mileDone = currentMileage - lastReplacedMileage`
- `milePercent = min(100, round(mileDone / maxMileage * 100))`

#### 8.2.5 关键交互
- HOME-001: 点击左箭头切上一辆车
- HOME-002: 点击右箭头切下一辆车
- HOME-003: 点击指示点切换到指定车辆
- HOME-004: 点击“添加换胎记录”跳记录页，携带 `vehicleId`
- HOME-005: 点击规格区域跳规格选择页
- HOME-006: 点击“一键预约”跳预约页，携带 `vehicleId`
- HOME-007: 点击知识“更多”跳知识页
- HOME-008: 点击单条知识跳知识页
- HOME-009: 点击“一键咨询”跳门店页
- HOME-010: 点击 AI 入口跳 AI 识别页
- HOME-011: 点击商品卡跳商品详情页
- HOME-012: 点击“一键购买”跳商品列表页

#### 8.2.6 状态要求
- 若没有车辆: 应展示无车空态（当前实现未完成）
- 若 `productStrip=conflict`: 应展示冲突提示

## 9. 当前实现与理想 PRD 的差距
- 缺少统一 loading skeleton
- 缺少统一 error 页面
- 缺少真正的 empty branch 覆盖
- 商品与部分视觉图仍混用外链资源
- 门店确认页只是返回上一页，没有把选择结果显式写回调用方
- 规格选择页暂未真正落回 vehicle 数据

## 10. v2 结论
本版已经足够支持:
- 按页面拆前端任务
- 按组件拆前端任务
- 做状态清单核查
- 做后端联调字段核对

但如果要用于“AI coding 直接写页面到最小颗粒度”，还需要继续下钻到更细的事件、文案、组件层和验收项，这将在 v3 完成。
